# PF6-3：一种基于ODD的水网自主运行能力分级测试方法

## （一）技术领域

本发明属于智慧水利自主控制验证技术领域，具体涉及一种面向CHS (Cybernetics of Hydro Systems) 与HydroOS平台的基于ODD (Operational Design Domain) 的水网自主运行能力分级测试方法，用于WNAL (Water Network Autonomy Level, L0-L5) 认证评估与升级验证。

## （二）背景技术

随着水网控制系统从人工辅助向自主运行演进，如何客观评估系统"能在何种运行域内安全自主工作"成为核心问题。现有项目中常以局部性能指标或单次示范运行作为验收依据，缺乏覆盖场景、性能达标与安全约束的统一分级测试标准。

水网自主能力受工况、设备健康、通信质量、外部扰动等多因素影响，若无明确运行设计域（ODD）定义，系统可能在超边界场景下仍维持高自主模式，导致风险累积。传统测试方法多依赖专家经验手工选场景，存在覆盖率不足、可重复性弱、等级划分主观等问题。

现有技术在自动驾驶和机器人领域已提出基于ODD的测试框架，但直接迁移到水网仍有障碍：第一，水网场景参数维度高且耦合复杂；第二，安全约束以连续过程变量为主，需要长时段评估；第三，自主等级与调度策略、联锁保护、降级机制紧密相关，单维指标无法充分刻画。

因此，需要一种面向水网的ODD分级测试方法，建立"场景覆盖率+性能达标率+安全零违规"的统一评估体系，为WNAL等级认证提供标准化依据。

| 对比维度 | 现有技术1：SAE J3016自动驾驶ODD测试框架 | 现有技术2：IEC 62443工控系统安全验证 | 本发明 |
|---------|------|------|--------|
| 适用对象 | 车辆自动驾驶(L0-L5) | 工业控制系统网络安全 | 水网自主运行(WNAL L0-L5) |
| ODD定义 | 道路/天气/交通场景 | 安全区域/通信协议 | 水力工况+水质+设备+环境+通信五维参数空间 |
| 分级标准 | 驾驶任务接管程度 | 安全等级(SL1-4) | 场景覆盖率+性能达标率+安全零违规三维评估 |
| 场景生成 | 基于道路拓扑 | 基于威胁模型 | 基于ODD参数空间分层采样+边界加密 |

## （三）发明内容

### 1. 技术问题

本发明要解决的技术问题是：针对水网自主运行等级缺乏标准化、可量化、可追溯的测试方法问题，提出一种基于ODD的分级测试方法，实现不同WNAL等级的客观认证与升级验证。

### 2. 技术方案

本发明提供一种基于ODD的水网自主运行能力分级测试方法，包括如下步骤：

**步骤S1：构建水网ODD参数空间。**

定义ODD向量：
$$
\Xi=[\Xi_f,\Xi_q,\Xi_m,\Xi_e,\Xi_c]
$$
其中 $\Xi_f$ 为流量与水位工况域，$\Xi_q$ 为水质域，$\Xi_m$ 为设备健康域，$\Xi_e$ 为环境扰动域，$\Xi_c$ 为通信与计算条件域。

**步骤S2：定义WNAL L0-L5分级测试要求。**

为每一级定义：
1) 必测场景集 $\mathcal{S}_l$；
2) 性能阈值集 $\Theta_l$；
3) 安全前提与降级规则 $\Gamma_l$。

各等级具体量化指标如下表：

| 等级 | 场景覆盖率阈值$C_l^{th}$ | 性能达标率阈值$P_l^{th}$ | 必测场景数下限 | 降级响应时间上限 | ODD复杂度要求 |
|------|------|------|------|------|------|
| L0(人工) | 不要求 | 不要求 | 0 | 不要求 | 不要求 |
| L1(监控辅助) | $\ge 0.70$ | $\ge 0.75$ | $\ge 30$ | $\le 300\,s$ | 仅正常工况 |
| L2(部分自主) | $\ge 0.85$ | $\ge 0.85$ | $\ge 80$ | $\le 120\,s$ | 正常+部分边界 |
| L3(条件自主) | $\ge 0.90$ | $\ge 0.90$ | $\ge 150$ | $\le 60\,s$ | 正常+边界+单一故障 |
| L4(高度自主) | $\ge 0.95$ | $\ge 0.93$ | $\ge 250$ | $\le 30\,s$ | 正常+边界+复合故障 |
| L5(完全自主) | $\ge 0.98$ | $\ge 0.96$ | $\ge 500$ | $\le 10\,s$ | 全ODD含未知扰动 |

所有等级均要求安全零违规率 $S_{safe}=1$ 为硬约束。

**步骤S3：执行场景采样与生成。**

在ODD参数空间内采用分层采样生成测试场景：
$$
\mathcal{S}=\{s_i(\Xi)\}_{i=1}^{N}
$$
对关键边界采用加密采样，对常规区域采用均匀采样，确保覆盖代表性与边界挑战性。

分层采样算法伪代码：

```
算法：ODD分层场景采样
输入：ODD参数空间Xi, 目标等级l, 场景数N, 边界加密倍率K_b
输出：测试场景集S

1. 将Xi划分为正常子域Xi_n、边界子域Xi_b、故障子域Xi_f
2. 根据等级l确定三类场景比例:
     ratio_n = {L1:0.80, L2:0.58, L3:0.47, L4:0.40, L5:0.35}[l]
     ratio_b = {L1:0.15, L2:0.25, L3:0.30, L4:0.30, L5:0.30}[l]
     ratio_f = 1 - ratio_n - ratio_b
3. N_n = floor(N * ratio_n), N_b = floor(N * ratio_b), N_f = N - N_n - N_b

4. // 正常子域：拉丁超立方均匀采样
5. S_n = LHS_sample(Xi_n, N_n)

6. // 边界子域：加密采样
7. 识别ODD边界面 B = boundary_faces(Xi)
8. FOR each 边界面b in B DO
9.     N_b_local = ceil(N_b * b.importance / sum(importance))
10.    S_b_local = stratified_sample_near_boundary(b, N_b_local, width=0.1*range(b))
11.    S_b = S_b UNION S_b_local
12. END FOR

13. // 故障子域：组合采样
14. 定义故障类型集 F = {传感器漂移, 执行器卡滞, 通信延迟, 模块掉线, ...}
15. IF l <= L3 THEN
16.    S_f = single_fault_sample(F, Xi_f, N_f)       // 单一故障
17. ELSE
18.    S_f = compound_fault_sample(F, Xi_f, N_f, max_compound=3) // 复合故障
19. END IF

20. // 可行性筛选
21. S_raw = S_n UNION S_b UNION S_f
22. S = empty
23. FOR each s in S_raw DO
24.    IF check_physical_feasibility(s) AND check_safety_constraint(s) THEN
25.        S.add(s)
26.    END IF
27. END FOR

28. // 覆盖率检查
29. IF coverage(S, Xi) < C_l^th THEN
30.    补充采样至满足覆盖率要求
31. END IF

32. RETURN S
```

**步骤S4：开展闭环测试执行。**

在MIL/HIL环境运行被测系统，记录控制响应、降级动作与安全联锁行为。每个场景输出评估结果：
$$
R_i=[P_i,S_i,V_i]
$$
其中 $P_i$ 为性能指标达标情况，$S_i$ 为安全约束结果，$V_i$ 为违规项。

**步骤S5：建立三维评估框架。**

定义等级评分：
$$
Score_l=\alpha C_{cov}+\beta P_{pass}+\gamma S_{safe},\quad \alpha+\beta+\gamma=1
$$
其中 $C_{cov}$ 为场景覆盖率，$P_{pass}$ 为性能达标率，$S_{safe}$ 为安全零违规率。

推荐权重取值：$\alpha=0.3$，$\beta=0.3$，$\gamma=0.4$，体现安全优先原则。

**步骤S6：等级判定与证据归档。**

当满足：
$$
C_{cov}\ge C_l^{th},\quad P_{pass}\ge P_l^{th},\quad S_{safe}=1
$$
则判定通过等级 $L_l$ 认证；否则输出未通过原因并给出整改建议。测试证据、日志和报告归档用于审计追溯。

### 3. 有益效果

1. 形成面向水网自主运行等级的标准化测试流程；
2. 通过ODD参数空间定义提高场景设计系统性；
3. 三维评估框架兼顾覆盖、性能与安全要求；
4. 支持等级升级前的量化门槛验证；
5. 为HydroOS自主运行认证提供统一证据链。

## （五）具体实施方式

### 实施例1：WNAL L2等级认证测试

选取某输水段自主控制系统，目标认证WNAL L2。构建ODD参数空间后生成测试场景120个，其中正常工况70个、边界工况30个、故障注入20个。测试在MIL平台执行，周期30 s，总时长72 h。

结果：
- 场景覆盖率 $C_{cov}=0.93$；
- 性能达标率 $P_{pass}=0.91$；
- 安全零违规率 $S_{safe}=1$。

满足L2门槛（$C_2^{th}=0.85$，$P_2^{th}=0.85$），判定通过。

### 实施例2：WNAL L3升级评估

在L2基础上引入更高复杂度场景（通信抖动、设备降额、扰动叠加）。共测试180个场景，发现12个场景在降级触发时延上不满足阈值（要求$\le 60\,s$），性能达标率降至0.86，未达到L3要求 $P_3^{th}=0.90$。

系统自动输出整改建议：优化降级判据参数、收紧边界触发阈值、补充联锁动作优先级。

### 实施例3：整改后复测与通过

完成策略整改后复测180个场景：
- $C_{cov}=0.95$；
- $P_{pass}=0.92$；
- $S_{safe}=1$。

达到L3认证阈值，系统输出等级证书与完整测试证据包。

### 对照实验：本发明ODD分级测试法与传统测试法对比

在同一输水工程上，分别采用传统专家经验测试法和本发明ODD分级测试法进行对照：

| 对比指标 | 传统专家经验测试法 | 本发明ODD分级测试法 |
|---------|------|--------|
| 场景设计依据 | 专家经验+历史案例 | ODD参数空间系统化采样 |
| 场景数量 | 25个(人工设计) | 120个(自动生成) |
| 边界场景覆盖 | 3个(经验选取) | 30个(边界加密采样) |
| 故障场景覆盖 | 2个(常见故障) | 20个(含复合故障) |
| 场景覆盖率 | 0.42(事后评估) | 0.93(在线计算) |
| 等级判定依据 | 专家打分(主观) | 三维量化指标+阈值判定 |
| 发现的薄弱点 | 2个 | 7个(含5个边界耦合问题) |
| 整改闭环 | 无系统化复测 | 自动输出整改建议+复测+重判 |
| 测试可重复性 | 低(依赖专家) | 高(参数化场景可精确复现) |
| 等级认证可信度 | 低(无量化依据) | 高(三维评估+证据链归档) |

结果表明，本发明方法在场景覆盖率上提升约121%，薄弱点发现数量提升约250%，且提供了可量化、可追溯的等级认证依据。

## （六）权利要求书

**权利要求1.** 一种基于ODD的水网自主运行能力分级测试方法，其特征在于，包括以下步骤：

步骤一，构建包含流量与水位工况域 $\Xi_f$、水质域 $\Xi_q$、设备健康域 $\Xi_m$、环境扰动域 $\Xi_e$ 和通信计算条件域 $\Xi_c$ 的五维ODD参数空间 $\Xi$；

步骤二，按WNAL L0-L5各等级定义必测场景集 $\mathcal{S}_l$、性能阈值集 $\Theta_l$ 和安全前提与降级规则 $\Gamma_l$，其中等级越高要求的场景覆盖率阈值 $C_l^{th}$、性能达标率阈值 $P_l^{th}$ 和必测场景数下限越高；

步骤三，在ODD参数空间内将参数空间划分为正常子域、边界子域和故障子域，分别采用拉丁超立方均匀采样、边界面加密采样和故障组合采样生成测试场景集 $\mathcal{S}$，并经物理可行性筛选；

步骤四，在MIL或HIL环境执行闭环测试，记录每个场景的性能指标 $P_i$、安全约束 $S_i$ 和违规项 $V_i$；

步骤五，基于三维评估公式 $Score_l=\alpha C_{cov}+\beta P_{pass}+\gamma S_{safe}$（$\alpha+\beta+\gamma=1$）计算等级评分；

步骤六，当 $C_{cov}\ge C_l^{th}$、$P_{pass}\ge P_l^{th}$ 且 $S_{safe}=1$ 时判定通过等级 $L_l$ 认证，否则输出未通过原因与整改建议。

**权利要求2.** 根据权利要求1所述的方法，其特征在于，所述安全零违规率 $S_{safe}=1$ 为所有等级通过的硬约束条件。

**权利要求3.** 根据权利要求1所述的方法，其特征在于，所述分级测试场景包括正常工况场景、边界工况场景和故障注入场景，三类场景比例随目标等级升高而调整，高等级中故障场景占比增加。

**权利要求5.** 根据权利要求1所述的方法，其特征在于，所述三维评估中推荐权重为 $\alpha=0.3$、$\beta=0.3$、$\gamma=0.4$，体现安全优先原则。

**权利要求6.** 根据权利要求1所述的方法，其特征在于，各等级降级响应时间上限随等级递减，L1不超过300 s，L3不超过60 s，L5不超过10 s。

## （七）摘要

本发明公开了一种基于ODD的水网自主运行能力分级测试方法。该方法构建包含工况、水质、设备、环境与通信条件的五维ODD参数空间，按WNAL L0-L5各等级定义量化的场景覆盖率阈值、性能达标率阈值和降级响应时间上限；通过分层采样生成测试场景，在MIL/HIL环境执行闭环测试并采集结果；基于场景覆盖率、性能达标率和安全零违规率建立三维评估框架，当覆盖率与达标率满足阈值且安全零违规时判定等级通过，否则输出未通过原因与整改建议并支持复测。该方法可为水网自主运行等级认证提供标准化、可量化、可追溯的测试依据。
