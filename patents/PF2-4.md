# PF2-4：一种水网自主运行分级评定与等级切换方法

## （一）技术领域

本发明属于水网运行管理与自动控制技术领域，具体涉及一种面向水网自主运行能力的分级评定与等级切换方法，尤其涉及水网自主运行等级 WNAL（Water Network Autonomy Level, L0-L5）的指标体系构建、等级判定与安全切换协议。本发明属于水系统控制论（CHS）八原理中层级原理（P8）和自适应原理（P7）的工程实现，为水网系统建立量化的自主运行能力评估与渐进式升级路径。

## （二）背景技术

随着水网从传统SCADA监控向自主运行升级，工程上迫切需要回答两个核心问题：第一，当前系统“自主到什么程度”；第二，何时可以安全升降级。现有项目多采用经验打分或单指标考核，缺乏统一等级定义和可执行切换规则，导致评估结果不可比、升级路径不清晰。

当前存在的主要不足包括：

1. **等级定义不统一**：不同项目对“自动化”“自主化”概念混用，难以跨工程横向比较；
2. **指标体系不完整**：常只看控制精度，忽略故障恢复时间、安全违规率、人机接管效率等关键指标；
3. **缺少切换协议**：即使完成评定，也缺少升降级触发门槛与切换过程约束，存在冒进升级风险；
4. **缺少运行闭环**：评定结果未反哺控制策略与运维流程，无法形成持续改进机制。

已有相关标准与研究对工业自动化分级、自动驾驶ODD分级等给出启发，但尚未形成适用于水网工程、可与HydroOS联动的WNAL分级与切换体系。

因此，需要一种可量化、可审计、可执行的水网自主运行分级评定与等级切换方法。

## （三）发明内容

### 1. 技术问题

本发明要解决的技术问题是：构建统一WNAL分级标准和多维评定指标体系，并给出可实施的等级升降级触发条件与安全切换协议。

### 2. 技术方案

本发明提供一种水网自主运行分级评定与等级切换方法，包括如下步骤：

**步骤S1：定义WNAL六级等级模型。**

- **L0**：人工主导，系统仅监测；
- **L1**：规则自动化，局部自动执行；
- **L2**：闭环自动调节，可处理常规扰动；
- **L3**：分层分布式协同控制，可在限定ODD内自主运行；
- **L4**：高自主运行，可在多数工况下自主决策并自动恢复；
- **L5**：全域自主运行，在定义ODD内无需人工实时干预。

**步骤S2：建立多维评定指标体系。**

构建性能、安全、韧性、可解释性四维指标：

- 性能维：操作准确率 $I_a$、响应时延 $I_t$、跟踪误差 $I_e$；
- 安全维：安全违规率 $I_s$、约束越界次数 $I_v$；
- 韧性维：故障恢复时间 $I_r$、降级成功率 $I_d$；
- 可解释性维：策略可追溯率 $I_x$、人机接管成功率 $I_h$。

综合评分为：

\[
S=\sum_{k=1}^{n} w_k\hat I_k
\]

其中 $\hat I_k$ 为归一化指标，$w_k$ 为权重，且满足 $\sum w_k=1$。

**步骤S3：等级判定规则。**

采用“总分+红线”双门槛：

- 当 $S$ 达到目标等级阈值且安全红线指标满足条件时，判定通过；
- 任一红线指标不达标则等级上限受限，触发整改。

示例红线：$I_s\le I_{s,max}$，$I_v=0$（关键工况）。

**步骤S4：等级切换触发条件。**

- **升级触发**：连续评估周期满足目标等级门槛，且关键场景覆盖率超过阈值；
- **降级触发**：发生红线违规、ODD越界、关键模块失效或恢复时间超限。

**步骤S5：安全切换协议。**

切换过程采用“五步协议”：预告→冻结参数→切换执行→稳态确认→审计留痕。  
切换期间优先执行安全约束：禁止超权限动作，必要时回退到低等级托底策略。

**步骤S6：评定-改进闭环。**

评定结果回写至HydroOS配置中心，自动生成“指标短板—整改任务—复评计划”闭环，实现持续升级。

### 3. 有益效果

1. 给出统一WNAL分级语言，提升跨工程可比性；
2. 采用多维指标与红线约束，避免”高分不安全”；
3. 提供可执行升降级协议，降低切换风险；
4. 支持审计留痕与闭环改进，提升长期治理能力；
5. 本方法是水系统控制论（CHS）八原理中层级原理（P8，分层分级管理）与自适应原理（P7，运行能力随经验积累渐进提升）在水网自主运行等级管理领域的工程化实现，为水网从人工主导向全域自主的演进提供了系统化理论支撑与可操作的实施路径。

## （五）具体实施方式

以下结合附图和具体实施例对本发明作进一步详细说明。

### 一、WNAL六级等级模型的详细定义

本发明所述水网自主运行等级（WNAL）分为L0至L5共六个等级，各等级的功能要求、性能指标门槛和安全前提如下。该分级体系借鉴了国际汽车工程师学会自动驾驶分级标准SAE J3016的核心思想，但针对水网运行的连续流体过程特征、长时滞大惯性动力学特性以及关键基础设施安全要求进行了系统性适配。

**L0——纯人工操作级。** 该等级下水网运行完全依赖人工操作，系统不提供任何自动控制功能。调度员通过现场巡视或电话联络获取运行信息，闸门启闭和水泵开停均由人工在现场或中控室手动执行。系统可能配备基础仪表但无远传能力，操作准确率完全取决于人员经验和技能水平。L0级适用于尚未完成自动化改造的老旧工程，其典型特征是无SCADA系统或SCADA系统仅用于事后记录而不参与实时决策。

**L1——远程监视级。** 该等级下水网系统具备远程监测能力，SCADA系统可实时采集并显示水位、流量、闸位等关键运行参数，调度员能够在中控室通过显示屏观察全网运行状态。然而，所有操作指令仍由人工下达，系统不提供自动控制建议。L1级的核心功能要求是数据采集完整率不低于95%、数据传输延迟不超过60秒、关键监测断面覆盖率不低于80%。L1级对应的人员配置要求是每个控制站必须有持证值班员在岗。

**L2——辅助操作级。** 该等级下系统具备初步的辅助决策能力，能够根据预设规则或控制算法计算推荐操作方案并呈现给调度员，但最终执行仍需人工确认。典型功能包括水位超限告警、建议调闸开度、推荐水泵组合方案等。L2级的性能指标门槛为：操作准确率（系统推荐方案被采纳且效果达标的比率）不低于95%，响应时延（从工况变化到系统给出推荐方案的时间）不超过30秒，关键工况安全违规率为零。L2级要求系统具备至少单渠池级的闭环控制能力，但控制指令下达前须经人工审批。当前我国多数已完成自动化改造的大型输水工程处于L1至L2过渡阶段，例如南水北调中线一期工程已实现全线SCADA监控和部分渠段的辅助决策功能。

**L3——条件自主级。** 该等级下系统能够在经过验证的运行设计域（ODD）内实现自主运行，无需人工逐一确认操作指令。控制器根据实时监测数据和预测模型自主计算并执行调控方案，仅在ODD边界逼近或异常事件发生时请求人工介入。L3级的性能指标门槛显著高于L2级：操作准确率不低于99%，响应时延不超过5秒，故障恢复时间（从检测到故障到系统恢复安全运行状态的时间）不超过15分钟，ODD覆盖率（已验证场景数占总运行场景数的比率）不低于85%，策略可追溯率（每条控制指令可回溯至决策依据的比率）不低于95%。L3级是水网自主运行的关键里程碑，要求系统完成分层分布式控制（HDC）架构部署，并通过系统化的在环测试（MIL、SIL、HIL）验证。根据南水北调中线工程十四五智能化升级规划，目标在2030年前实现核心渠段达到L3级自主运行能力。

**L4——高自主级。** 该等级下系统能够处理大多数运行场景，包括部分超出原始ODD边界的非典型工况。L4级系统具备ODD自适应扩展能力，可通过在线学习和数字孪生预演将新场景纳入已验证运行域。其性能指标门槛为：操作准确率不低于99.5%，响应时延不超过2秒，故障恢复时间不超过5分钟，ODD覆盖率不低于95%。L4级要求部署完整的多智能体系统（MAS），各层级Agent具备自主决策与协商能力，通信中断时边缘Agent可独立维持安全运行。L4级系统允许减少值班人员配置，但仍须保留远程监督岗位。

**L5——全域自主级。** 该等级下系统在定义的全部运行域内实现完全自主运行，无需人工实时干预。系统具备自主扩展ODD的能力，能够自动识别新场景、在数字孪生环境中验证应对策略、经安全评估后纳入正式运行域。L5级的性能指标门槛为：操作准确率不低于99.9%，响应时延不超过1秒，故障恢复时间不超过2分钟，ODD覆盖率不低于99%，人机接管成功率（当人工选择接管时系统顺利交接控制权的比率）不低于99%。L5级代表水网自主运行的终极目标，当前尚无实际工程达到该等级。

### 二、等级评定指标体系与计算方法

本发明建立覆盖四个维度共九项指标的评定体系。各维度及其指标的具体定义如下。

性能维包含三项指标。操作准确率 $I_a$ 定义为评估周期内自主控制指令使系统状态到达目标区间的比率，L2级要求 $I_a \ge 0.95$，L3级要求 $I_a \ge 0.99$，L4级要求 $I_a \ge 0.995$。响应时延 $I_t$ 定义为从工况变化被检测到控制指令到达执行器的时间，L2级要求 $I_t \le 30$ 秒，L3级要求 $I_t \le 5$ 秒，L4级要求 $I_t \le 2$ 秒。跟踪误差 $I_e$ 定义为实际水位与目标水位偏差的均方根值，以渠池设计水深的百分比表示。

安全维包含两项指标。安全违规率 $I_s$ 定义为评估周期内出现水位越限、流量越限等安全违规事件的频率，L3级及以上要求 $I_s = 0$（关键工况零违规）。约束越界次数 $I_v$ 为绝对计数指标，任何等级均要求关键工况下 $I_v = 0$。

韧性维包含两项指标。故障恢复时间 $I_r$ 定义为从故障发生到系统恢复至安全运行区间的时间，L3级要求 $I_r \le 15$ 分钟，L4级要求 $I_r \le 5$ 分钟。降级成功率 $I_d$ 定义为触发降级条件时系统在规定时间内完成等级切换且未发生二次故障的比率。

可解释性维包含两项指标。策略可追溯率 $I_x$ 定义为每条控制指令可回溯至明确决策依据（模型预测值、优化目标函数值、触发规则编号等）的比率。人机接管成功率 $I_h$ 定义为人工发起接管请求后系统在规定时间内完成控制权平稳移交的比率。

综合评分按加权归一化方法计算：$S=\sum_{k=1}^{9} w_k\hat I_k$，其中各维度权重推荐配置为性能维0.35、安全维0.30、韧性维0.20、可解释性维0.15。等级判定采用”总分达标且红线无一触碰”的双门槛机制，即综合分 $S$ 达到目标等级阈值（L2阈值0.65，L3阈值0.75，L4阈值0.85，L5阈值0.95）且所有红线指标均满足要求时方可认定达到对应等级。

### 三、等级切换协议

等级切换包括升级切换和降级切换两类，二者的触发条件和执行流程存在本质差异。

**升级切换协议。** 升级是渐进式的审慎过程，须满足以下条件方可启动：连续三个评估周期（每周期不短于30天，共计至少90天）的综合评分均达到目标等级阈值；所有红线指标在全部三个评估周期内无一违反；ODD关键场景覆盖率达到目标等级要求。满足上述条件后，系统进入为期90天的影子运行试用期。在影子运行期间，待升级功能模块在后台独立运行并产生控制指令，但该指令不直接下达给执行器，而是与当前等级控制器的实际指令进行实时比对。当影子运行期间的性能指标持续满足目标等级要求且与实际控制器的指令偏差在允许范围内时，经工程负责人审批后方可正式执行升级切换。切换过程严格遵循五步协议：预告（提前24小时通知所有相关人员和系统）→冻结参数（锁定当前运行参数作为回退基准）→切换执行（按预定计划切换控制模式）→稳态确认（切换后持续监测至少72小时确认系统稳定）→审计留痕（自动生成完整的切换过程报告）。

**降级切换协议。** 与升级不同，降级是安全驱动的即时响应，一旦触发条件成立须立即执行。降级触发条件包括：发生红线违规（安全违规率超标或关键工况约束越界）；ODD边界距离指标低于安全阈值（即当前运行工况接近或超出已验证运行域）；关键控制模块失效（如通信中断、传感器故障、控制器异常）；故障恢复时间超过当前等级的规定限值。降级切换无需审批流程，由系统自动执行。以L3降级至L2为例，系统立即将自主控制模式切换为辅助操作模式，所有控制指令暂停自动下达并改为推荐方案供调度员确认，同时向值班人员发出接管告警。降级过程中优先执行安全保护策略，必要时进一步降级至L1甚至L0，确保任何情况下安全底线不被突破。降级完成后系统自动生成事件报告并创建复评任务，记录降级原因、持续时间和影响范围，为后续恢复升级提供依据。

### 四、与自动驾驶SAE J3016等级的类比与差异

本发明的WNAL分级体系在设计理念上借鉴了SAE J3016自动驾驶分级标准，但在技术内涵上存在重要差异。自动驾驶面向的是离散决策场景（加速、制动、转向），而水网控制面向的是连续流体动力学过程，控制周期从毫秒级安全保护到日级调度计划横跨多个时间尺度。自动驾驶的ODD以道路类型、天气条件等离散类别定义，而水网的ODD以水位范围、流量范围、水质指标、气象条件、设备状态等连续参数空间定义。此外，水网作为关键基础设施，其安全裕度要求远高于单车驾驶，等级升级过程更为审慎。因此，本发明在L3至L5各等级的性能门槛设定上均严于SAE J3016对应等级的通行做法。

### 五、实施例

#### 实施例1：区域输水工程WNAL等级评定

选取某含34座闸站、全长260公里的区域输水网络作为评定对象。采用30天滚动窗口进行指标采集，权重配置为性能维0.35、安全维0.30、韧性维0.20、可解释性维0.15。经过一个评估周期的数据采集与分析，测得各归一化指标如下：操作准确率 $\hat I_a = 0.87$，响应时延 $\hat I_t = 0.82$，跟踪误差 $\hat I_e = 0.79$，安全违规率 $\hat I_s = 1.0$（零违规），约束越界次数 $\hat I_v = 1.0$（零越界），故障恢复时间 $\hat I_r = 0.75$（实测12分钟，门槛15分钟），降级成功率 $\hat I_d = 0.80$，策略可追溯率 $\hat I_x = 0.72$，人机接管成功率 $\hat I_h = 0.68$。

按加权公式计算综合得分 $S = 0.35 \times (0.87+0.82+0.79)/3 + 0.30 \times (1.0+1.0)/2 + 0.20 \times (0.75+0.80)/2 + 0.15 \times (0.72+0.68)/2 = 0.35 \times 0.827 + 0.30 \times 1.0 + 0.20 \times 0.775 + 0.15 \times 0.70 = 0.289 + 0.300 + 0.155 + 0.105 = 0.849$。该得分超过L3阈值0.75，同时安全红线指标全部满足（关键工况零越界、零违规），因此判定该系统当前已达到WNAL-L3等级。评定报告同步回写至HydroOS配置中心，作为后续升级至L4的基线。

#### 实施例2：升级切换验证（L3向L4升级）

在上述区域输水网络中验证L3向L4的升级切换流程。在连续三个30天评估周期中，系统综合分分别为0.86、0.88和0.87，均超过L4阈值0.85。然而第二周期中检测到一次ODD边界逼近事件（某渠段来水量达到设计流量的92%，接近已验证ODD上限95%）。按照本发明的升级协议，该次ODD边界逼近虽未触发降级（未实际越界），但升级决策须进入”条件观察加参数收紧”状态。系统自动将该渠段的ODD安全裕度阈值从5%收紧至8%，并延长观察一个评估周期。第四周期综合分0.89，无任何ODD边界逼近事件。随后进入90天影子运行期，L4控制策略在后台运行并与L3控制器实际指令比对，偏差率始终低于3%。经工程负责人审批，按五步协议执行正式升级：预告全网相关人员→冻结L3参数作为回退基准→切换至L4控制模式→72小时稳态监测确认→生成升级审计报告。整个切换过程未发生安全违规。

#### 实施例3：异常降级验证（L3向L2降级）

模拟通信主链路中断叠加一台闸门执行器液压故障的复合场景。通信中断导致协调优化层（DMPC）无法与相邻渠池交换边界流量轨线，触发关键控制模块失效条件。同时执行器故障导致该闸门无法响应控制指令，故障恢复预估时间超过L3规定的15分钟限值。系统自动触发降级切换，按五步协议在90秒内完成L3向L2切换：立即冻结当前控制参数→将自主控制模式切换为辅助操作模式→向值班调度员发出接管告警→相邻渠池Agent自动切换至独立运行模式（岛式自主）→确认所有渠池水位均在安全区间内。降级期间，边缘Agent依据本地IDZ模型维持渠池水位在安全范围内，保障了安全底线不被突破。故障排除后，系统自动生成降级事件审计报告并创建复评任务，包含降级原因分析、影响范围评估和恢复升级时间计划。

## （七）摘要

本发明公开了一种水网自主运行分级评定与等级切换方法。该方法定义WNAL L0-L5六级自主运行等级，建立覆盖性能、安全、韧性和可解释性的多维评定指标体系，采用“综合得分+安全红线”双门槛完成等级判定，并通过升级/降级触发条件与五步安全切换协议实现等级平稳切换。评定结果可回写HydroOS配置中心，形成“评定—整改—复评”持续改进闭环。实施结果表明，本方法可有效提升水网自主运行能力评估的一致性、可审计性和工程可实施性。
